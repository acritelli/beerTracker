About Beer Tracker

Beer Tracker is designed to provide a way to track beer consumption at the 2017 Rochester Real Beer Expo. It was created by Anthony Critelli

Answers to potential questions:

Why not just use Untappd?

Untappd is awesome! I just find it tedious to try searching for beers while I'm at beer tasting events. I usually end up carrying around a slip of paper to take notes on (and see others doing the same). Having a simple interface to quickly jot down notes and rate my favorite beers will hopefully make this easier. Plus, you can export the notes to an Excel file and manually enter them into Untappd later. It's definitely not seamless, but I personally prefer it to looking up every single brew during the event.

Can it do x,y,z?

Probably not, but I'd like to actually improve upon this (assuming people use it). Feel free to email your suggestions to critellia AT gmail dot com.

What's the technology stack?

The code is on Github. Some additional info:

Backend:

  • Python - uses the Flask framework
  • MongoDB - let me focus on quickly building the app, since I didn't have to put much thought into data design (i.e. let me be lazy).

Frontend:

  • HTML
  • CSS - Bootstrap
  • Javascript - jQuery

Why does the code suck?

Yeah...it's really, really rough. This was written in like 2 days (spontaneous Saturday decision the week before the expo), and I'm not a web developer. The code definitely needs a lot of work, but I wanted to get something at least working for the event, and I have other stuff going on.

My goal is to improve on this before the Flour City Brewers Fest. Refactoring and design work are all part of the plan. If you have suggestions, please feel free to email them to me: critellia AT gmail dot com.
